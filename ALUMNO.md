## ¿Qué es Git?

Git es un sistema de control de versiones que permite administrar y registrar los cambios realizados en archivos de un Proyecto. donde asi mismo facilita el trabajo en equipo al permitir que varias personas trabajen de manera simultanea sin afectar el avance de los demas.

## Por qué necesitamos Git

Porque es crucial porque permite organizer el trabajo en equipo de una manera eficiente, evitando la Perdida de informacion y conflictos entre versiones donde puede llegar a ser un muro en otros sistemas. Sin Git, sería difícil controlar los cambios realizados por diferentes personas y mantener un historial ordenado del proyecto.  

## Los 3 conceptos clave de Git

1. **Repositorio:** Es el espacio donde se almacena el proyecto junto con todo su historial de cambios. Donde este mismo puede estar de forma local o en la nube.
2. **Commit:** Es el registro de un cambio realizado en el proyecto, el cual guarda una versión específica con un mensaje descriptivo.
3. **Rama:** Es una línea de desarrollo independiente que permite trabajar en nuevas funciones sin afectar la versión principal del proyecto.

## Los comandos que aprendí

git init - Inicializa un repositorio nuevo
git status - Muestra el estado de los archivos
git add - Agrega archivos al área de preparación
git push - Sube los cambios realizados a GitHub 
git commit - Guarda los cambios realiazados

## Mi reflexión final

Lo más importante que aprendí de Git es la importancia de mantener un control sobre las versiones de un proyecto. Git facilita el trabajo en equipo, mejora la organización y reduce errores, lo cual es esencial en proyectos de desarrollo de software.
